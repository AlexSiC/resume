Кейс 01 / Видеобаза знаний

Видеобаза знаний для инженерной компании

Система, которая превращает зум-встречи, испытания оборудования и производственные видео в searchable knowledge base: загрузка через Telegram или браузер, автоматическая транскрибация, RAG-поиск — и ответ, подкреплённый фрагментами транскриптов, на которых он основан.

02 / Проблема

Инженерные знания были заперты внутри видео

В компании накапливались сотни часов записей: зум-встречи, апробации линий, испытания оборудования, разборы нестандартных ситуаций. Формально данные сохранены — практически ими невозможно пользоваться: чтобы найти нужный фрагмент, приходилось пересматривать часы видео.

  • Невозможно быстро найти, где обсуждался конкретный узел, дефект или решение
  • Знания уходят вместе с сотрудниками
  • Видеоархив не участвует в ежедневной работе
  • Ручная разметка, теги и регламенты не приживаются
03 / Ключевая идея
Вход в систему должен быть таким же простым, как отправить видео в Telegram или загрузить файл в браузере.

Главный принцип — не заставлять сотрудников менять поведение. Никаких форм, обязательных тегов и регламентов: дальше оба сценария входа работают через один пайплайн обработки.

Вход A

Telegram-бот

Нулевой порог входа: снял видео — отправил в чат. Умеют все.

Вход B

Браузерный интерфейс

Управляемая загрузка: пакеты файлов, метаданные, просмотр и редактирование базы.

05 / Сценарий

Как это выглядит для инженера

  1. Инженер снимает видео испытания или получает запись встречи
  2. Отправляет файл в Telegram-бота — или загружает через браузер
  3. Система сохраняет видео, транскрибирует речь и индексирует содержание
  4. Позже инженер спрашивает: Где обсуждали проблему с датчиком на линии?
  5. Система возвращает подготовленный ответ и подборку фрагментов транскриптов, которые лучше всего отвечают на вопрос

Что я сделал

  • Реализовал Telegram-бота для загрузки видео
  • Реализовал браузерный интерфейс загрузки и управления базой

  • Подготовил слой RAG-поиска

07 / Результат

08 / Позиционирование

Это не Telegram-бот. Это инфраструктура знаний.

  • Бот и браузерный интерфейс — только удобные точки входа; ценность — в пайплайне обработки данных
  • Система связывает видео, текст, метаданные и поиск в один контур
  • Архитектура рассчитана на большие файлы, фоновые задачи и LLM-поиск
  • Решение адаптируется под производство, обучение, техподдержку и аудит

Большие видео из Telegram

Фоновая обработка

Очереди задач: приём, хранение больших файлов, транскрибация и индексация — без блокировки пользовательских сценариев.

Подготовка данных для RAG

Из сырых транскриптов — в чанки и индекс, по которым находится конкретный ответ, а не «где-то в этом часе записи».

UX для нетехнических сотрудников

Единый пайплайн за двумя простыми точками входа: никакого обучения и регламентов.
